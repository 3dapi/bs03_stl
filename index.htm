◈STL(Standard Template Library)◈


1. 개요

STL은 템플릿 자료 구조 + 제너릭 알고리듬으로 표현이 됩니다. 템플릿은 자료 구조를 담고 있는 그릇이라 할 수 있습니다. 프로그램에서는 자료의 구조보다는 자료를 처리하는 방식에 더 관심이 있을 때 템플릿을 사용하게 됩니다.

자료의 정렬은 순서대로 원소를 순차적으로 구성하거나 나무(Tree) 형태, 또는 키 값을 가지고 구성할 수 있습니다. 그런데 이러한 자료의 정렬의 알고리듬을 살펴보면 원소 구조에 상관 없이 일정한 자료에 대한 접근 방식을 따르고 있음을 볼 수 있습니다. 즉, C++ 언어의 템플릿으로 구성할 수 있습니다.

STL은 Standard Template Liberty의 약어 입니다. 위에서 정렬, 찾기, 기타 전산학에서 사용되는 일반적인 자료의 처리들은 제너릭 알고리듬에 포함되어 있습니다. STL은 원소(element:데이터)의 처리에 대한 알고리듬 뿐만 아니라 원소를 포함하고 있는 컨테이너로 구성되어 있습니다.

컨테이너는 크게 원소의 구성에 따라 순차 컨테이너, 연관 컨테이너로 구분합니다. 순차 컨테이너(Sequential Container)는 배열이나 연결 리스트(Linked List)처럼 순서대로 접근해 나가도록 원소를 구성하는 컨테이너로 게임 프로그램에서는 벡터(vector), 리스트(list), 데크(deque) 컨테이너들을 주로 사용합니다.

빠른 검색을 목적으로 하거나 원소의 중복을 제한하기 위해서 키 값을 가지고 간접적으로 접근할 수 있도록 원소들을 구성하는 컨테이너를 연관 컨테이너(혹은 연상 컨테이너: Associated Container)라 합니다. 연관 컨테이너 중에서 집합처럼 유일성을 요구될 때 사용되는 셋(Set)과 멀티 셋(Multi Set), 빠른 검색을 위해서 키와 값의 쌍으로 구성되어 검색 속도를 높인 맵(map)과 멀티 맵(Multi-Map)을 게임프로그램에서 주로 사용합니다.

STL은 컨테이너 요소들의 순회에서 일반화된 객체를 이용하는데 이것을 반복자라 합니다. 이 반복자는 마치 포인터처럼 행동을 하기도 해서 기억하기 좋게 포인터의 일반화라 생각해도 좋지만 반드시 포인터는 아닙니다. 이 반복자는 Generic Algorithm에서 많이 이용되므로 사용법을 잘 숙지하는 것이 좋습니다.

 

2. 주요함수

STL 컨테이너들은 동일한 역할을 수행하는 함수들은 동일한 이름으로 사용합니다. 컨테이너 사이에 같은 이름을 가진 함수들 중에서 중요 함수들을 살펴보면 다음과 같습니다.

 

1. push_back: 원소를 컨테이너의 끝에서 추가하며 벡터와 같은 순차 컨테이너에서 빈번하게 사용되는 함수 중의 하나입니다.

2. insert(): 원소를 중간에 추가하는데 사용되는데 키를 이용한 자료 구조로 구성된 연관 컨테이너에서 사용됩니다.

2. pop_front(), pop_back(): 원소를 맨 앞 또는 맨 뒤에서 꺼내올 때 사용하며, 원소를 꺼내오고 나서 컨테이너에서 이 요소를 삭제 합니다.

3. clear(): 컨테이너에 들어있는 원소를 비우는데 사용되며 컨테이너 자체 또한 지워집니다.

4. empty(): 컨테이너에 원소가 있는지 확인합니다. size로도 확인할 수 있지만 원소에 대한 유무를 확인 하는 것이라면 이것을 사용하는 것이 좋습니다.

5. size(): 컨테이너의 논리적 크기이며 이것은 실제로 컨테이너에 포함되어 있는 원소의 개수와 일치 합니다. 배열의 개수를 조사하는데 사용됩니다.

6. begin(): 컨테이너의 첫 번째 요소의 반복자를 반환합니다. 반복자는 포인터처럼 동작하는데 이 begin() 함수는 시작 반복자를 가리킵니다. 특히 벡터(vector) 이외의 컨테이너에서는 begin()함수를 통해서 논리적인 연산에 따라 원소를 접근해야 하므로 가장 빈번 하게 사용되는 함수 중 하나입니다.

7. end(): 컨테이너의 마지막 원소의 다음 위치에 있는 반복자를 가져오는 함수입니다. 컨테이너는 마지막 원소의 끝에 컨테이너의 끝임을 표시하는 end 반복자를 하나 더 가지고 있습니다. 반복자로 컨테이너의 원소 탐색, 정렬, 추가, 삭제 등에서 begin()함수와 더불어 구간 설정에 많이 사용됩니다.

 

 

3. 벡터(Vector)

3.1 사용법 기초

배열은 모든 자료 구조 중에서 원소에 대한 임의 접근이 가장 빠릅니다. 그런데 배열은 한 번 정해지면 동적으로 크기를 조절하기가 어렵습니다. STL에 중에서 벡터는 배열처럼 원소의 접근 속도를 빠르게 접근 할 수 있습니다. 또한 배열과 다르게 크기를 조절할 수 있게 만든 순차 컨테이너입니다.

벡터는 배열처럼 인덱스로 원소를 접근하도록 [] 연산자(operator)를 다중 정의(Overloading)되어 있어서 다른 컨테이너와 달리 임의 접근이 가능합니다. 또한 [] 연산자를 통해서 원소를 접근해도 배열과 비교했을 때 그 속도가 거의 비슷하다고 보고되고 있습니다. (참고: Efficient C++)

벡터는 원소의 추가(push_back), 삭제(pop_back)가 편리해서 사용이 유연하고 제너릭 알고리듬과 결합해서 사용하는데 편리한 점이 많이 있어서 게임 프로그램에서 가장 많이(다른 컨테이너 사용과 비교해서 거의 90% 이상) 사용되는 STL 컨테이너라 할 수 있습니다.

 

먼저 간단히 벡터를 연습해 보도록 합시다.

 

#include <vector>

#include <stdio.h>

 

void main()

{

        …

        std::vector< int >     vInt;

        …

        for(i=0; i< 13; ++i)

               vInt.push_back(i);

        …

        if(!vInt.empty())

        {

               for(i=0; i< vInt.size(); ++i)

                       printf("%d\n", vInt[i]);

        }

       

        vInt.clear();

}

 

위의 예제는 정수형 원소를 가진 벡터 컨테이너에 10개의 원소를 추가하고 [] 연산자를 통해서 원소를 가져오고 있습니다. 또한 벡터를 사용하고 나서 마지막에 컨테이너를 clear() 함수를 통해서 비웁니다.

 

이외에 [] 연산자를 다중 정의 하는 방법을 알고 싶다면 다음 예제를 참고하기 바랍니다.

<반복자와 포인터>

반복자는 STL에서 원소의 순회에 대한 일반화라 이야기 했습니다. 이 반복자를 사용하면 좀 더 일반적인 프로그램 작성이 가능합니다. 다음은 반복자를 사용해서 원소를 접근하고 해제하는 예제입니다.

 

#include <vector>

#include <stdio.h>

 

struct GameData

{

        int nIdx;

        int data;

       

        GameData(int _nIdx, int _data)

        {

               …

        }

};

 

void main()

{

        …

        std::vector<GameData* >       pvData;

       

        for(i=0; i< 10; ++i)

               pvData.push_back( new GameData(i, i*100) );

 

        std::vector< GameData* >::iterator    _F = pvData.begin();

        std::vector< GameData* >::iterator    _L = pvData.end();

       

        for(; _F != _L; ++_F)

        {

               printf("%d  %d\n", (*_F)->nIdx, (*_F)->data);

        }

 

        _F = pvData.begin();

       

        for(; _F != _L; ++_F)

               delete (*_F);

        …

}

 

반복자 사용에 대해서 몇 가지 기억해야 할 것은 첫 번째로 for(; _F != _L; ++_F) 와 같이 ++ 연산자 혹은 – 연산자를 통해서 순차적으로 원소를 접근하는 것입니다. [] 연산자는 벡터와 맵 컨테이너에만 정의 되어 있습니다. 하지만 단 항 증가 연산자 ++은 모든 컨테이너에 전부 정의 되어 있습니다. 따라서 반복자와 ++연산자를 이용한다면 STL의 컨테이너에 대해서 보편적인 코드를 작성할 수 있습니다.

다음으로 반복자는 원소의 주소와 동등한 역할을 한다는 것을 꼭 기억해야 합니다. 초보자들이 쉽게 혼동하는 부분이 이 부분인데 원소를 가져오기 위해서 (*반복자)->, 또는 (*반복자).을 사용하고 있음을 기억하기 바랍니다. 위의 예제와 같은 것을 이 글을 읽는 독자께서 한 번 예제를 만들어 본다면 앞으로 어려움을 없으리라 생각 됩니다.

<erase>

erase()함수는 컨테이너 원소를 삭제하는 함수 입니다. 앞서 원소가 new와 같은 연산자로 Heap으로 할당한 경우 컨테이너를 clear하기 전에 메모리를 해제해야 한다고 했습니다. 마찬가지로 ease() 함수로 원소를 삭제하는 경우에도 먼저 Heap을 해제하고 erase() 함수를 호출해야 합니다.

 

또한 ease()함수를 호출하면 컨테이너의 end가 달라 집니다. 따라서 특정조건에 맞는 데이터를 삭제를 하려면 매 번 컨테이너의 반복자를 end와 비교해야 합니다. 다음은 원소를 new로 생성한 후에 100의 배수가 아닌 원소들을 지우는 예입니다.

 

struct MyClass

{

…

};

 

 

void main()

{

        typedef vector<MyClass*>      lsMyClass;

        typedef lsMyClass::iterator itMyClass;

        lsMyClass vMyClass;

 

…

        for(_F = vMyClass.begin(); _F != vMyClass.end(); )

        {

               if( 0 != (*_F)->nIdx%100)

               {

                       vMyClass.erase(_F);

                       continue;

               }

 

               ++_F;

        }

…

}

 

ease() 함수는 원소를 삭제하게 되므로 end 가 변경이 되고, 또한 반복자를 증가시킵니다. 보통 원소를 검색할 때 for(_F = vMyClass.begin(); _F != vMyClass.end(); ++_F) 로 하거나 begin, end의 원소를 얻어 for(; _F != _L; ++_F) 과 같이 작성하는 경우가 대부분이지만 ease()함수를 사용하는 경우라면 매 번 end를 검사하는 것이 좋습니다. 또한 삭제한 경우 continue를 사용하고, 반복자의 증가는 for 문 끝에서 처리 하는 것이 좋습니다.

3.2 정렬

정렬에서 기억 장치 사용과 정렬의 속도에서 퀵 정렬(Quick Sort)가 가장 우수하다고 알려져 있습니다. C 라이브러리에는 퀵 정렬을 담당하는 qsort 함수가 있습니다. 사용법은 다음과 같습니다.

 

#include <stdio.h>

#include <malloc.h>

 

struct TGameData

{

        int nIndex;

 

        static int Greater( const TGameData* v1, const TGameData* v2)

        {

               if( v1->nIndex > v2->nIndex)          return -1;

               else if( v1->nIndex == v2->nIndex)    return 0;

               else                                  return 1;

        }

};

 

void main()

{

        …

        TGameData*     pN = (TGameData*) malloc( iN * sizeof(TGameData));

        …

        qsort(pN, iN, sizeof(TGameData)

3.3 함수자 사용

단순하게 위와 같은 qsort() 함수 대신 제너릭 알고리듬에 있는 sort() 함수를 사용해서 정렬을 해도 되지만 좀 더 ‘제너릭’ 하게 만들기 위해서 함수자(Functor)를 사용하는 것이 일반적입니다.

함수자(Functor)는 함수 객체(Function Object)를 뜻하며 함수의 역할을 수행하는 객체를 지칭합니다. 함수자는 순전히 함수에 대한 동작 만을 처리합니다. 만약 "()연산자"가 다중 정의(Overloading) 정의되어 있다면 함수자가 될 수 있습니다. 일반적으로 C++ 에서는 다음과 같이 함수자를 구현합니다.

return …;

}

5. Set & Multi-Set

 

셋(Set:집합) 컨테이너는 집합과 같은 원소의 유일(unique)한 자료를 처리하는데 사용됩니다. 예를 들어 집합에서 A = {a,a,a,a, b,b,b}와 같은 것은 없고 A = {a, b}와 같은 모습이 올바를 형태 입니다.

이렇게 집합처럼 원소의 유일성을 보장하므로 게임에서는 중복된 원소 추가를 제한하는데 사용됩니다. 셋과 멀티-셋(Multi-set)은 원소 자체가 처음부터 2진 나무(tree)로 정렬 되어 있는 상태로 유지됩니다. 또한 셋은 중복된 원소를 제거하지만 멀티-셋은 정렬만 유지할 뿐입니다.

셋과 멀티-셋 컨테이너에서 주의할 점은 미리 함수자를 넣어서 정렬 방식을 미리 정해야 합니다.

6. Map

Map은 대표적인 연관 컨테이너의 표본입니다. 원소의 탐색에서 키를 이용한 검색이 이 컨테이너의 목적입니다. Map은 마치 해쉬 테이블(Hash Table)처럼 원소를 바로 찾아갈 수 있는데 셋이 원소의 유일성에 초점이 있다면 Map은 빠른 탐색에 초점이 있습니다.

 

맵은 하나의 키(Key)와 값(value)로 구성되어 있는데 흥미로운 점은 [] operator입니다. [] 연산자는 키에 대한 값을 바로 전달해 주는데 예를 들어 키가 "Hello"로 되어 있고 값이 1000이라고 한다면 맵은 이것을 마치 스크립트에서 사용하는 것처럼 a["Hello"] 와 같은 방식으로 원소를 찾아올 수 있도록 [] 연산자를 정의해 놓았습니다.

다음 예제는 Map에 원소를 추가하고, 찾는 예제입니다.
